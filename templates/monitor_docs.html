Alert Severity Levels

Severity Color Response Time Example Events
Critical Red Immediate (1-5 min) Data exfiltration, System breach
High Orange Urgent (15-30 min) Policy violation, Unauthorized access
Medium Yellow Normal (1-2 hours) Suspicious activity, Failed login attempts
Low Blue Informational (24 hours) System notifications, Regular scans

Real-time Metrics

  • Active Threats: Currently detected security threats
  • Policy Violations: Count of policy breaches
  • User Activities: Real-time user actions monitoring
  • System Health: Resource usage and status
  • Compliance Status: Regulatory compliance metrics
  • Network Traffic: Data transfer monitoring
  • Scan Activities: Active and completed scans

Alert Response Workflow

  1. Detection: System identifies potential threat
  2. Classification: Assigns severity level
  3. Notification: Alerts security team
  4. Investigation: Security team reviews alert
  5. Action: Takes appropriate security measures
  6. Resolution: Marks alert as resolved
  7. Documentation: Logs incident details
Tip: Configure alert thresholds and notification preferences in Settings. Regular review of alerts helps identify patterns and improve security policies.

API Endpoints for Monitoring

GET /api/alerts # List all alerts
GET /api/alerts/active # Active alerts only
GET /api/alerts/{id} # Specific alert details
PUT /api/alerts/{id}/resolve # Resolve an alert
GET /api/metrics/realtime # Real-time metrics
GET /api/metrics/historical # Historical data
POST /api/monitor/config # Update monitoring config
